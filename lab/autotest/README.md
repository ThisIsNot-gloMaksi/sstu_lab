# Автотест лабы

Чёрный ящик: дёргает запущенную сагу по HTTP и проверяет поведение. Один и тот
же тест работает и для оркестрации, и для хореографии.

## Установка

```bash
pip install -r ./lab/autotest/requirements.txt
```

## Запуск

1. Подними нужный вариант саги (с добавленным сервисом `antifraud`):

   ```bash
   cd orchestration   # или choreography
   docker-compose up --build
   ```

2. В другом терминале из корня репозитория (команда одинаковая для обоих вариантов):

   ```bash
   ORDER_URL=http://localhost:8000 \
   ANTIFRAUD_URL=http://localhost:8010 \
   pytest -v ./lab/autotest/test_saga.py
   ```

Тест запускай на **свежеподнятом** стеке (склад стартует с `INITIAL_STOCK=5`).
Проверка компенсации опирается на этот объём — если меняешь `INITIAL_STOCK`,
передай то же значение тесту переменной `INITIAL_STOCK`.

## Что проверяется
1. `order` и `antifraud` отвечают на `/health`.
2. Обычный заказ проходит антифрод и завершается (`COMPLETED`, decision `passed`).
3. Подозрительный заказ (сумма > `FRAUD_LIMIT`) отклоняется (`FAILED`, decision `rejected`).
4. **Компенсация склада**: после отклонения антифродом зарезервированный товар
   возвращается — легитимный заказ на тот же объём снова проходит. Проверяется
   чёрным ящиком, поэтому работает и в оркестрации, и в хореографии (где склад
   без HTTP).
